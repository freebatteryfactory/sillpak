# Visual and interaction direction

## Principle

This is an instrument panel for real work, not an AI landing page embedded inside a desktop window.

## Spatial grammar

- top: identity, route, and dictation destination
- left: compact workspace orientation
- center: route-owned artifact page
- bottom: persistent terminal
- right: explicit context, only when space permits

## Materials

Use typography, white space, baseline alignment, hairline separators, and restrained surface contrast.

Buttons state their action in text. File kinds use short extension marks or native operating-system icons when Electron can supply them. There is no icon library.

## Explicitly rejected patterns

- gradient backgrounds
- glass blur
- glowing focus halos everywhere
- rounded cards around every section
- generic “AI assistant” orb or chat bubble
- dashboard KPI tiles unrelated to the task
- sidebar icon rail copied from code editors
- fake terminal decoration
- unexplained sparkle icons
- Lucide icon monoculture
- huge marketing headlines inside the work surface

## Motion

Motion communicates route continuity and panel state only. It respects reduced-motion and should never delay terminal or file interaction.

## Platform manners

The shell keeps one spatial grammar but adopts platform shortcuts, menu placement, file operations, and window behavior. Do not manufacture one visual smoothie from Explorer, Finder, and Linux file managers.
